# Hard Rules

Inviolable constraints for all agents. Violations are blocking errors.

## Code Standards

- **Max 500 lines per file.** Split into modules if approaching limit.
- **No raw SQL.** All data access goes through Payload Local API exclusively. No direct Drizzle queries.
- **Never edit `payload-types.ts` manually.** It is auto-generated by `payload generate:types`.
- **`payload.config.ts` lives at project root.** Not in `src/`, not in `config/`.
- **All public APIs must have typed interfaces.** No `any` types in exported functions or components.
- **TypeScript strict mode.** `strict: true` in `tsconfig.json`. No exceptions.

## Block Naming and Structure

```
src/blocks/{BlockName}/
  config.ts        # Payload block schema definition
  Component.tsx    # React render component
```

- Block folder names: **PascalCase** (e.g., `Hero`, `CallToAction`, `RichContent`)
- Block slug in config: **camelCase** (e.g., `hero`, `callToAction`, `richContent`)
- Every block must have both `config.ts` and `Component.tsx`. No orphaned files.

## Tenant Isolation (CRITICAL)

- **Factory agents** must NOT access files outside the assigned tenant repo.
- **Digital Team agents** must NOT read or write the Golden Image repo.
- **No cross-tenant database access.** Each tenant has its own SQLite file in its own container.
- **No cross-tenant Caddy routing modification.** Agents may only modify routes for their assigned tenant.
- **No shared state between tenants.** No shared Redis, no shared DB tables, no shared file paths.

## Security

- **Never commit `.env` files or secrets.** Use `.env.example` for templates.
- **Never hardcode API keys, tokens, or credentials** in source files.
- **Validate all input at system boundaries.** Payload hooks for CMS data, middleware for API routes.
- **Sanitize file paths** to prevent directory traversal in any file operation.

## Process

- **All changes must pass `npm run verify` before committing.** Verify = lint + typecheck + test.
- **One session = one task = one branch.** Branch naming: `feature/{task-name}` or `fix/{task-name}`.
- **Never force-push to `main`.** All changes to `main` go through PR + verification.
- **No client components without justification.** Default to React Server Components. Add `'use client'` only when browser APIs or event handlers are required.

## File Organization

- Source code: `src/`
- Blocks: `src/blocks/`
- Collections: `src/collections/`
- Globals: `src/globals/`
- Components: `src/components/`
- Context docs: `docs/context/`
- Skills: `docs/skills/`
- Scripts: `scripts/`
- Tests: `tests/` or co-located `*.test.ts` files

## Forbidden Actions

- Do not install packages without adding them to `package.json` dependencies.
- Do not create circular imports between blocks.
- Do not use Payload REST API when Local API is available (server-side context).
- Do not store media files in the git repo. Use R2 storage adapter.
- Do not modify Docker container runtime config from inside the container.
